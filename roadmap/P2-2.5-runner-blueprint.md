### 3.2 Universal handler shape
```python
def <stage>_handler(req: dict, d: StageDeps) -> dict:
    with job_workspace(req) as ws:                       # TemporaryDirectory, auto-cleanup
        in_paths = download_inputs(d.r2, req, ws)        # validate each logical key, stream GET
        t0 = perf_counter()
        out_paths = <core>(in_paths, req["params"], ws, d)   # core + seams; fail-closed
        refs = upload_outputs(d.r2, req["outputPrefix"], out_paths)  # stream PUT, sha256+bytes
        metrics = {"duration_ms": round((perf_counter() - t0) * 1000), **counters}
        validate_metrics(metrics)                        # numbers-only assert (fatal if not)
        return {"outputs": refs, "metrics": metrics}
```
- `download_inputs` raises `ValueError` if a required logical key is absent in `req["inputs"]` (a wiring bug is not retryable → fatal `VALUE_ERROR`).
- `upload_outputs` streams SHA-256 (`artifacts.sha256_file`, 1 MiB chunks, never `f.read()`) and returns `ArtifactRef` dicts `{key, bytes, sha256}` — **always all three** (§5 covers the lax-optional contract).
- Outputs uploaded ONLY after the core verified them (render fail-closed + atomic-rename gate). An unverifiable output raises before any upload.

## 4. Content-hash idempotency

**No table. No sentinel in Python. No DB.** (Authority = Node `execute-stage.ts:24` sentinel + existing `upload_ledger` PK=content_hash.) Python guarantees the three properties that make Node's authority crash-safe:

1. **Deterministic content-addressed keys** (`content_key`) → a retried/duplicate compute writes the same keys; R2 PUT is idempotent per key (last-writer-wins). Two workers racing the same `(stage, contentHash)` PUT byte-identical artifacts — harmless.

2. **Crash-safe local artifact materialization** — `render_vertical_clips` orchestration writes `clip_NNN.mp4.partial` (same dir → same fs → atomic rename), runs the EXISTING size+probe checks **on the partial**, then `_replace_fn(partial, final)` (default `os.replace`). The canonical path appears only when fully verified. A SIGKILL mid-encode leaves a `.partial` (swept by `TemporaryDirectory`), never a truncated canonical mp4. This is the local mirror of "write artifact THEN commit marker" — the marker (sentinel) is written by Node only after Python returns ok.

3. **Integrity record without server-side attrs** — `upload_outputs` streams SHA-256 and embeds it in every `ArtifactRef.sha256` so Node persists it in the sentinel body + `clips` row.

**Crash-safe ordering across the boundary (proven):** Python uploads all artifacts → returns `{ok, outputs}` → Node `executeStage:24` writes the sentinel LAST. Crash between upload and sentinel → no sentinel → next run recomputes, overwrites orphans (safe). Crash mid-upload → no sentinel, partial keys overwritten on retry (safe). The sentinel never precedes the artifacts.

**R2 client config** (`stages/r2.py` `build_config`, pure & tested):
```python
Config(
    request_checksum_calculation="when_required",   # botocore≥1.36 — R2 breaks otherwise
    response_checksum_validation="when_required",
    signature_version="s3v4", region_name="auto",   # R2 literal "auto"
    retries={"max_attempts": 5, "mode": "adaptive"},
    connect_timeout=10, read_timeout=120, max_pool_connections=20,
)
TransferConfig(multipart_threshold=64*MiB, multipart_chunksize=64*MiB,  # equal parts ≥5MiB R2 floor
               max_concurrency=4, use_threads=True)
```
`download_file`/`upload_file` stream to/from disk. `check_part_limit(size)` (pure) raises `ValueError` if `size > 10000 * 64 MiB`. `is_missing_key(ClientError)` (pure) → `True` on `Error.Code in {NoSuchKey, NoSuchBucket}` or HTTP 404. In `download_file` (pragma'd I/O): `except ClientError as e: raise ValueError(...) if is_missing_key(e) else raise` — 404 → fatal, transient 5xx/timeout → ClientError → retryable (via §5 explicit classification).
