## 1. Визуальное направление

**Направление: Dark AI-tech.** Не «чистый минимализм по умолчанию», а намеренно тёмная, глубокая, «продуктовая» эстетика уровня Linear / Vercel / Lovable / v0.

### 1.1 Ключевые качества (минимум 4 из чек-листа design-quality на каждой значимой поверхности)

1. **Иерархия через контраст масштаба** — огромный hero-заголовок (`--text-hero` clamp до 8rem) против сдержанного body. Никакого «всё одного размера».
2. **Глубина и слоистость** — фоновый WebGL mesh-gradient (GPU-слой) → стеклянная панель дропзона (glass + blur) → парящие чипы вложений. Реальные тени, оверлапы, свечение (glow).
3. **Атмосфера / зерно** — анимированный grain/noise поверх градиента, чтобы убрать «пластиковую» гладкость и дать киношную фактуру.
4. **Семантический цвет** — accent (violet) и accent-2 (cyan) несут смысл (фокус, активное состояние, статус отправки), а не просто украшают.
5. **Designed-состояния** — hover/focus/active/drag-active у дропзона и кнопки отправки спроектированы явно (свечение бордера, смена иконки стрелки по статусу).
6. **Моушен, проясняющий поток** — scroll-driven раскрытие секций, scrubbed pin, плавный Lenis-скролл; движение объясняет структуру, а не отвлекает.

### 1.3 Анти-шаблон

Запрещено: дефолтные карточные сетки с равномерным паддингом, стоковый центрированный заголовок + градиентный блоб + дженерик CTA, немодифицированные дефолты библиотек, плоские layout без слоёв и движения, серое-на-белом с одним декоративным акцентом. FlipHouse должен выглядеть как реальный продукт на скриншоте, а не как Tailwind/shadcn-стартер.

## 3. Сборка центрированного hero-дропзона (Lovable-style)

**Решение:** `AI Elements PromptInput` как ОБОЛОЧКА + `Kibo Dropzone` внутри как drop-поверхность + контролируемый textarea для paste-link. Всё — copy-owned shadcn-исходники, без runtime SDK-lock-in.

### 3.1 Слои (back → front)

```
[GPU-слой]  shadergradient MeshGradient (absolute inset-0), speed≈0.3, встроенный grain
              ↓ (под prefers-reduced-motion → speed=0, статичный кадр)
[Атмосфера] Magic UI / Aceternity Aurora glow + BorderBeam вокруг панели
[Панель]    glass-поверхность (--color-glass, backdrop-blur, radius 24px, тень + glow)
[Контент]   огромный H1 (--text-hero) → PromptInput с Kibo Dropzone → чипы вложений
```

Установка:
```bash
npx ai-elements add prompt-input      # оболочка
npx kibo-ui add dropzone              # drop-поверхность
npm i @shadergradient/react @react-three/fiber three   # фон (code-split, ssr:false)
npm i motion                          # движок моушена
```

### 3.2 Логика «drop + paste-link + send» в одном боксе

1. **Состояние:** `files: File[]`, `link: string`, `status: 'ready'|'submitted'|'streaming'|'error'`.
2. **Drop:** `<Dropzone accept={{'video/*':[]}} maxFiles={1} maxSize={500*1024*1024} src={files} onDrop={setFiles} onError={()=>setStatus('error')}>` — `<DropzoneEmptyState/>` авто-подпись «Upload up to N files… up to X MB», `<DropzoneContent/>` — имена/превью. `isDragActive` сам тоглит active-стиль.
3. **Paste-link:** контролируемый `PromptInputTextarea`/input на `link`; на paste — URL-regex; при матче видео-URL показать «link-чип» рядом с file-чипом, чтобы оба инпута визуально сосуществовали.
4. **Send + статус:** `<PromptInputSubmit status={status}/>` внутри `<PromptInput onSubmit={(msg)=>{ if(!files.length && !isUrl(link)){setStatus('error');return;} setStatus('submitted'); kickOffFlip({file:files[0], url:link}); }}>`. Во время обработки — `streaming`, по завершении → `ready`/`error`. Иконка стрелки/спиннер/ошибка рендерятся компонентом сами.
5. **globalDrop** на `PromptInput`: видео, брошенное в любом месте hero, попадает в бокс.

> Альтернатива, если центрированный инпут — load-bearing требование и нужен реальный рабочий multimodal-input с drag-drop: поднять `multimodal-input` + `suggested-actions` из `vercel/ai-chatbot` (Apache-2.0, сохранить attribution).

## 4. Система oklch-токенов

**Пайплайн (3 слоя, один генерируемый файл):**

```
tweakcn (визуальный авторинг)  →  tokens/*.json (SOURCE OF TRUTH)
        →  npm run tokens (Style Dictionary, color/oklch transform)
        →  src/styles/tokens.css  (:root/.dark, потребляется globals.css @theme inline)
        +  COLORS.md  (таблица контрастности — «что видно по цветам»)
        →  [опц.] npm run tokens:registry  →  registry-item.json (npx shadcn add)
```

### 4.1 Слой 1 — PRIMITIVES (тема-независимые рампы)
`--oklch-violet-{50..900}`, `--oklch-cyan-{50..900}`, neutral-рампа. Сырые oklch-стопы.

### 4.2 Слой 2 — SEMANTIC (имена shadcn + raw-сигналы) в `:root` — Swiss Pop, светлая база
```css
:root {
  --background:            oklch(97.5% 0.004 95); /* paper */
  --foreground:            oklch(17% 0.012 270);  /* ink */
  --card:                  oklch(97.5% 0.004 95);
  --card-foreground:       oklch(17% 0.012 270);
  --primary:               oklch(63% 0.244 26);   /* vermillion */
  --primary-foreground:    oklch(16% 0.03 26);    /* on-pop-solid (AA on vermillion) */
  --secondary:             oklch(48% 0.210 258);  /* cobalt */
  --secondary-foreground:  oklch(99% 0.004 95);
  --ring:                  oklch(63% 0.244 26);   /* vermillion focus */
  --border:                oklch(86% 0.006 270);  /* hairline rule */
  --muted:                 oklch(94% 0.004 95);
  --muted-foreground:      oklch(40% 0.012 270);
  /* raw signals (match the reference var names for a direct P1.5 port) */
  --pop:                   oklch(63% 0.244 26);
  --cobalt:                oklch(48% 0.210 258);
  --ink-soft:              oklch(40% 0.012 270);
  --ink-faint:             oklch(62% 0.010 270);
  --rule:                  oklch(86% 0.006 270);
  --rule-strong:           oklch(17% 0.012 270);
}
```
Тёмная тема не проектируется (светлая база). Источник правды — `apps/web/tokens/*.json`.

### 4.3 Слой 3 — NON-COLOR (общие)

## 5. План моушена (только compositor-friendly)

**Жёсткое правило:** анимировать ТОЛЬКО `transform`, `opacity`, `clip-path`, `filter` (умеренно). НИКОГДА — `width/height/top/left/margin/padding/border/font-size`. `will-change` ставить узко и снимать после.

### 5.1 Hero

| Элемент | Эффект | Свойства | Источник |
|---|---|---|---|
| Фон | анимированный mesh-gradient + grain | GPU canvas (вне layout) | shadergradient |
| Aurora/glow | мягкое свечение за панелью | `opacity`, `transform` | Magic UI / Aceternity |
| H1 | text-reveal по словам при появлении | `transform: translateY`, `opacity`, `clip-path` | motion-primitives `TextEffect` / Aceternity Text Generate |
| Панель | BorderBeam + появление | `transform: scale`, `opacity` | Magic UI BorderBeam |
| Drop-state | drag-active свечение бордера | `box-shadow`/`filter`, `transform` | Kibo `isDragActive` |
| Submit | смена иконки + спиннер по статусу | `opacity`, `transform: rotate` | AI Elements `PromptInputSubmit` |

### 5.2 Scroll-driven секции

- **Движок:** GSAP + ScrollTrigger (dynamic import per perf-rules), Lenis для плавного скролла.
- **Синк:** `lenis.on('scroll', ScrollTrigger.update)` + `lenis.raf` в `gsap.ticker`.
- **Паттерны:** pinned hero reveal (scrub timeline), clip-path section reveals (изучить `adrianhajdin/award-winning-website` как reference), image-sequence scrub для «как работает FlipHouse» (паттерн из `adrianhajdin/iphone`), editorial text-reveal pacing (reference `olivierlarose`).
- **Декларативно (если React-first):** `basementstudio/scrollytelling` `Root`/`Animation`/`Pin`.

### 5.3 Reduced-motion и производительность

- `prefers-reduced-motion`: shadergradient `speed={0}` (статичный кадр), Lenis-smoothing off, отключить scrub-таймлайны и pointer-driven fluid.
- Все WebGL/Three.js-канвасы: `next/dynamic ssr:false` / `React.lazy`, вне critical render path (защита LCP/FCP), пауза по `IntersectionObserver` вне вьюпорта, `dpr` cap + пониженный resolution на мобиле (INP/TBT).
- Бюджет лендинга: JS < 150kb gzip, CSS < 30kb. CWV-цели: LCP < 2.5s, INP < 200ms, CLS < 0.1, FCP < 1.5s, TBT < 200ms.
- Heavy-таймлайны и GSAP — `await import('gsap')` / `await import('gsap/ScrollTrigger')`, не в критическом пути.
