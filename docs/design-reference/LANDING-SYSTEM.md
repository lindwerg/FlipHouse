# FlipHouse — Система лендинга (строить 1:1)

> Канонический рулбук для всех публичных страниц FlipHouse. Визуальный эталон: [`swiss-pop.html`](./swiss-pop.html). Направление — **Swiss-Pop**: светлая paper/ink основа + vermillion/cobalt акценты, Archivo + IBM Plex Mono, **выключка влево** (flush-left). Любая новая страница (тарифы, кейсы, лендинги под аудитории) должна следовать этим правилам дословно, чтобы серия выглядела единой.
>
> Зафиксировано после полной зачистки лендинга (де-англизация, выравнивание, тема заработка, чистка футера, удаление AI-тире, идеальный hero, SEO/OG). Дата: 2026-06-15.

---

## 0. Принципы

1. **Весь видимый текст — на русском.** Латиницей остаются только бренды/форматы: `FlipHouse`, `YouTube`, `Vimeo`, `Zoom`, `Riverside`, `MP4`, `MOV`, `ShipFast`.
2. **Тон — про заработок** (см. [[positioning-money-first]]). Каждая секция связывает продукт с доходом автора, а не только с техникой. Честно: маркетплейс платит, но конкретную сумму не обещаем.
3. **Flush-left всегда.** Никаких центрированных колонок. Симметрия = строгие общие линии выравнивания и единый ритм, а не зеркальное центрирование.
4. **Конкретика убивает «водянистость»** — цифры и факты (12×, 9:16, 94%, $7 689) сохраняем.

## 1. Копирайт: словарь замен (англицизмы → русский)

В видимом тексте англицизмы **запрещены**. Замены:

| Англицизм | Русский |
|---|---|
| speaker-tracking / speaker-tracked | удержание спикера в кадре / кадрирование за спикером |
| реврейм / reframe | перевод в вертикаль / вертикальный кадр / перекадрирование |
| karaoke-субтитры | субтитры по словам / пословные субтитры |
| скор / скоры виральности | оценка виральности / балл виральности |
| impressions | показы |
| CPM | за 1000 показов |
| флэт-фи / flat fee | фиксированная оплата / фикс |
| медиакит | рекламная презентация / без презентаций для брендов |
| DM | личные сообщения / личные переписки |
| шипить / шипнуть | выпускать / выкладывать / публиковать |
| скроллить | листать / прокручивать |
| хук | зацепка / цепляющее начало |
| How-to / туториал | инструкция / пошаговый разбор |
| оффер | предложение (рекламодателя) |
| питч | обещания / презентация / «на словах» |
| креатор | **автор** (держим ОДИН вариант на весь продукт) |

**Оставляем:** `маркетплейс`, `шортсы`, `залетают` (одобренный нативный сленг), `конверсии`, `экспорт`. Текст внутри рекламного баннера ShipFast («ДЕПЛОЙ В 1 КЛИК») — диегетический креатив вымышленного рекламодателя, не голос FlipHouse, не переводим.

## 2. Тире и AI-почерк (оригинальность)

**Длинное тире (—): максимум одно на секцию, в идеале ноль в коротких подписях.** Правила:
- Тире между двумя мыслями → две фразы с точкой. («Загрузите видео — режет моменты» → «Загрузите видео. Режет моменты.»)
- Тире-пояснение/перечисление → двоеточие или запятая.
- Тире в mono-подписях/слоганах → удалить, переструктурировав.
- **Оставить** тире можно: в диапазонах таймкодов (`00:41 — 00:43` — это данные) и в смысловой паре «вход → выход». Стрелки `→` (кнопки) — это не тире, остаются.
- В заголовках-разделителях используем **двоеточие**, не тире (`FlipHouse: одно видео…`).

**Запрещённые AI-приёмы и штампы:**
- Конструкция «не X, а Y» как приём (максимум 1 раз на страницу). Переписываем в утверждение.
- Парные тире-вставки внутри предложения.
- Канцеляризмы-отглагольные («осуществляет нарезку») → живые глаголы («режет, кадрирует, переводит»).
- Вводные пустышки («Стоит отметить», «В современном мире»).
- Механические триплеты («быстро, удобно, надёжно»).
- Эмодзи/декоративные значки (`●`) как «оживление».
- Штампы: `раскрой потенциал`, `новый уровень`, `под ключ`, `инновационный/революционный`, `меняет правила игры`, `бесшовный`, `мощный инструмент`, `гарантированный доход`.
- Разнообразить длину предложений (рубленые + один длинный).

## 3. Денежный словарь (вплетать)

«Деньги на выходе» (закреплённый H1-акцент), «зарабатывать на том, что уже публикуешь», «оплата за результат», «платят за каждый клип», «строка выручки под каждым клипом», «выплата автору», «оплата за показы и конверсии», «превратите старые клипы в источник дохода», «добавь маркетплейс и начни зарабатывать», «от загрузки до выплаты». **Без обещаний конкретной суммы** (цифра $7 689 — помеченное демо).

## 4. Сетка и выравнивание

**Контейнер (каждая секция, идентично):** `mx-auto w-full max-w-[1600px] px-[var(--space-margin)]`.
**Каждая внутренняя сетка:** `grid grid-cols-12 gap-[var(--space-gutter)]`. Другой gutter между колонками не вводить.

**Три канонические линии колонок:**
- **Контент/заголовок** — col 1, `md:col-span-7` (большой заголовок клозера может быть span 8–9).
- **Номер секции** — col 1, `md:col-span-2` (только SectionHead).
- **Aside/вторичное** — **всегда `md:col-start-9`** (`md:col-span-4`). Hero-лид, aside в SectionHead, поддержка в Closer — все стартуют с col 9, чтобы правая колонка повторялась по всей странице. Исключение: парная figure в Marketplace — `lg:col-span-6 lg:col-start-7`.

**Правила:**
- Заголовки/контент **никогда не отступают** от левого края (всегда col 1).
- `md:text-right` разрешён только для aside в SectionHead и колонки оценки в RankedBatch. Больше ничего не right-align.
- **Колонки с вертикальными разделителями** (ProcessSteps, Metrics, 4-up) — текст НЕ прижимать к линии. Паттерн: `lg:border-r lg:px-6 lg:first:pl-0 lg:last:border-r-0`. Первая колонка на левой линии сетки, остальные с внутренним отступом от разделителя.

**Иерархия линий — ровно два веса:**
- **HEAVY** `border-[1.5px] border-[var(--rule-strong)]` — разделители секций, дропбар, figure маркетплейса, нижний руль SectionHead, верхний руль 4-up полос. **Все межсекционные границы — HEAVY.**
- **HAIRLINE** `border border-[var(--rule)]` — внутренние строки/ячейки (строки таблицы, низ ячеек полос, под-руль футера).

## 5. Вертикальный ритм

Существующие токены (`tokens.css`, не перетюнивать): `--space-section` (py секций), `--space-gutter` (gap колонок), `--space-margin` (боковые поля контейнера).

**Лестница шагов** (только из этого набора для внутренних отступов): `2, 3, 4, 6, 8, 10, 12, 16`. Mobile→md повышаем ровно на одну ступень (`mb-8 md:mb-12`, `mt-8 md:mt-10`, `py-6 md:py-8`). Не изобретать значения вне лестницы.

- **Секция:** `py-[var(--space-section)]`. Hero-исключение: `pt-12 md:pt-16` + `pb-[var(--space-section)]`. Closer: свой `py-[clamp(4rem,3rem+5vw,8rem)]` (тёмная финальная полоса) — единственное допустимое исключение.
- **SectionHead → контент:** `mb-8 md:mb-12` (живёт в SectionHead, не дублировать).
- **4-up ячейки (ProcessSteps + Metrics — идентично):** `py-6 md:py-8 pr-6` + отступы от линий (см. §4).

## 6. Hero (эталон)

- **H1 (единственный на странице):** `Видео на входе. Деньги на выходе.`, `<AnimatedHeading accentIndices={[3]}>` (слово «Деньги» — `--pop`). Не менять текст/акцент. `id="hero-h"`, `col-span-12`, font-grotesk font-black `leading-[0.9] tracking-[-0.035em]`, `fontSize: var(--text-hero)`.
- `--text-hero` потолок **8rem** (floor 2.4rem) — чтобы заголовок + дроп-зона помещались в первый экран. Не повышать.
- **Первичное действие сразу под H1:** HeroDropzone в первом экране. Над ним один eyebrow «Начните здесь. Загрузите видео.».
- **Лид справа на уровне дроп-бокса:** строка действия — `grid grid-cols-12 md:items-center`; дропбокс `md:col-span-7`, лид `md:col-span-4 md:col-start-9 md:mt-0`. Лид центрируется по высоте дропбокса (в его границах, не выше/не ниже).
- **Удалено навсегда:** mono-строка форматов (`● MP4 · MOV …`) — главный template-tell.
- Проверка идеального hero: один фокусный H1 ✓, одно действие над фолдом ✓, одна поддерживающая фраза ✓, нет декоративной строки форматов ✓, единый акцент `--pop` ✓.

---

## 7. Акценты (цвет)

`--pop` (vermillion) — единственный акцент: номер секции, шкала оценки rank-01, одна денежная фраза болдом, CTA, акцентное слово в H1. **`--cobalt` — только** для стороны «Для рекламодателей» в маркетплейсе (семантика двух аудиторий). Больше cobalt нигде.

Mono-лейблы: `tracking-[0.18em]` — для hero/section eyebrow; `tracking-[0.14em]` — для inline-лейблов таблиц/карточек. Ровно два значения по роли.

---

## 8. Футер

- Колонки на `grid-cols-12`: wordmark `md:col-span-4`, ссылки `md:col-span-2`.
- **Только живые ссылки** (реальные якоря `#batch #process #marketplace #pricing` или будущие маршруты). Заглушки на `#top`, «Вакансии» — **запрещены**. Если в колонке <2 живых ссылок — убрать колонку.
- **Колофон-строка запрещена** (`© … собрано на 12-колоночной сетке`, `Archivo · IBM Plex Mono — выключка влево`). Только одна минимальная строка: `© 2026 FlipHouse`, mono `0.74rem`, `text-[var(--ink-faint)]`, hairline-руль сверху.

## 9. SEO и Open Graph

- `<html lang="ru">` для русского контента (см. `resolveHtmlLang` в `layout.tsx`).
- `metadataBase` через `getBaseUrl()`. На странице — `alternates.canonical` + `languages` (hreflang) + `x-default`.
- Title/description/OG/twitter — на русском, **по тому же словарю** (без англицизмов и тире), title-разделитель — двоеточие. JSON-LD `WebApplication`.
- **OG-картинка** (`opengraph-image.tsx`): русская, зеркалит hero. Кириллица — через подгрузку Archivo (Google Fonts `&text=` сабсет, старый User-Agent форсит TTF для Satori), с try/catch-фолбэком. В проде задать `NEXT_PUBLIC_APP_URL` для абсолютных URL.

## 10. Hard-constraints вёрстки (не ломать)

- Marketplace LEDGER-строки: `flex flex-wrap items-baseline justify-between`. Figure full-width до `lg` (`col-span-12 lg:col-span-6 lg:col-start-7`), inner `items-start`, заголовок телефон-бокса `[overflow-wrap:anywhere]`.
- RankedBatch: адаптивный заголовок `Вир./Виральность`, шаблон ROW не трогать.
- Не вводить горизонтальный overflow. Проверять на 320/375/768/1024/1280/1440 (0px overflow, дроп-зона в фолде).
- Не менять `id`/`aria-*`/`data-reveal*`/`data-slot`/сигнатуры пропсов.
